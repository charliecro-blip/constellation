# Agent Workflow

- Run the test suite with `pytest -q` before finishing a feature batch.
- Never commit API keys, geocoding keys, or any other secrets.
- Do not add compatibility scoring, match verdicts, or deterministic fate claims.
- Preserve the Relationship Field Map product doctrine in code and copy.
- Prefer coherent feature batches over scattered unrelated edits.
- Update tests and docs whenever behavior changes.
- If calculation logic changes, add or update tests and/or validation fixtures.
- Keep frontend implementation simple unless there is a clear need for a build step.
