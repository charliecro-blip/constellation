# Astrology Doctrine

Constellation is a relationship-signature ranking system, not a compatibility-score app. The deterministic astrology layer calculates charts, synastry, overlays, composites, and pattern evidence; the doctrine decides which signatures deserve narrative priority in a Relationship Field Map.

The goal is to make strong relationship themes legible without turning astrology into verdicts. A high-ranking pattern means “lead with this in the report,” not “good,” “bad,” “fated,” or “doomed.”

## Doctrine flow

1. **Pattern registry** assigns each detected signature a category, tier, base weight, default report section, and lead eligibility.
2. **Scoring weights** define base tier strength, suppression thresholds, orb/aspect/body/house emphasis, and convergence multipliers.
3. **Weighting** adjusts detected patterns for relationship context, repeated motifs, double whammies, and synastry/composite echoes.
4. **Report opening rules** choose a lead-eligible pattern rather than the numerically loudest minor detail.
5. **Voice and repair guidance** keeps reports psychologically precise, non-fatalistic, and agency-preserving.
6. **Asteroid gating** keeps minor points narrow by default so they support rather than hijack the reading.
7. **Synthesis packets** expose the deterministic ranking and selected motifs to report construction and optional AI enhancement.
8. **Motif persistence** stores compact recurring relationship themes without preserving clutter.
9. **Diagnostics** show which signatures were ranked, suppressed, or surfaced.
