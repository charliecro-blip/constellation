# Report Prioritization Doctrine

Constellation ranks patterns internally to decide narrative order. It does not publish a compatibility score and should not imply that a higher score means a better relationship.

## Inputs to ranking

- **Pattern registry:** Each signature has a category, tier, base weight, lead eligibility flag, and default section.
- **Tier/base weights:** Tier 1 signatures start with the strongest report priority; lower tiers require context or convergence to rise.
- **Orb weighting:** Tight contacts are more narratively reliable than wide contacts of the same type.
- **Planet and angle significance:** Luminaries, Venus, Mars, Saturn, and the Ascendant/Descendant axis carry more relationship weight than peripheral bodies.
- **House relevance:** 1st, 4th, 5th, 7th, and 8th houses are relationship-central; 10th/MC themes need public-life context to lead.
- **Convergence multipliers:** Repeated categories, double whammies, natal echoes, and synastry-plus-composite confirmation increase confidence.
- **Lead eligibility:** The report opening should come from a category that can responsibly carry the central story.
- **Suppression thresholds:** Weak or unsupported details should be omitted rather than cluttering the report.

## Contextual exceptions

Communication signatures can move up when the user asks about communication, conflict, misunderstanding, or conversation. Public-life signatures can move up when the context names career, vocation, collaboration, visibility, reputation, or shared work.

## Output discipline

Ranking is an internal editorial tool. Reports should surface the few strongest themes, synthesize convergent evidence, and omit patterns below threshold. A clean report with fewer grounded themes is better than a long report that treats every contact as equally important.
