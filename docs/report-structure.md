# Relationship Field Map Structure

This document describes the target report structure for Constellation.

The current code generates only a first-pass draft report. This structure is the next target.

## Core Sections

### 1. Context
Relationship type, status, user question, known themes, and any origin story or symbolic context.

Purpose:
- avoid generic output
- distinguish romantic, family, friendship, collaboration, and admired-figure maps
- give the report a human situation

### 2. Bird's-Eye View
A short synthesis before technical detail.

Purpose:
- lead with the relationship thesis
- avoid dumping aspects before meaning
- name the largest field pattern

### 3. Person A Relational Profile
Natal relationship style for Person A.

Inputs:
- Moon
- Venus
- Mars
- Mercury
- Ascendant / chart ruler
- 7th house / 7th ruler
- Juno when available
- Saturn / Chiron / Uranus / Pluto relationship signatures

### 4. Person B Relational Profile
Same as Person A.

### 5. Mutual Activation / Synastry
How each person activates the other.

Core question:
> What part of me becomes louder in contact with you?

### 6. Desire & Affection Layer
Romantic, sensual, erotic, creative, and attraction signatures.

Important distinction:
- affection is not the same as desire
- desire is not the same as safety
- intensity is not the same as sustainability

### 7. Emotional Safety Layer
Moon contacts, Saturn contacts, 4th-house/IC material, Cancer/Capricorn themes, and emotional regulation styles.

Core question:
> How does the bond seek safety, and where does safety become difficult?

### 8. Communication Layer
Mercury contacts, Mercury/Mars, Mercury/Saturn, Mercury/Pluto, Mercury signs, 3rd-house themes.

Core question:
> How do words move between these people?

### 9. Partnership Pattern
Juno, 7th house, Descendant, Venus, Saturn, nodes, and long-term bond imagination.

Core question:
> What kind of partnership does each person imagine, and what kind of partnership does the relationship field ask for?

### 10. The Field Between You / Composite
Composite chart as relationship-being.

Core question:
> What does the relationship become as its own living pattern?

### 11. Surface vs Engine
Distinguish presentation from deeper field mechanics.

Examples:
- friendly surface / Plutonian engine
- airy communication / watery bonding system
- Cancer care / weak repair structure
- Sagittarius heat / limited home-building capacity

### 12. Friction Loop
A step-by-step description of how the relationship can get caught.

Purpose:
- make difficult signatures practical
- avoid doom statements
- show sequence and repair entry point

### 13. Repair Path
A practical principle or phrase.

Examples:
- Ask for rhythm before definition.
- Return to warmth before precision.
- Keep the freedom, reduce the ambiguity.
- Cooling down is not abandonment.

### 14. Reflection Prompts
Simple questions the user can use.

### 15. One-Sentence Summary
Memorable summary line.

## Product Rules

- No compatibility score.
- No good/bad verdict.
- Begin with synthesis.
- Interpret exact contacts before loose volume.
- Composite is required for relationship-being.
- Each hard pattern needs a repair path.
- The map cannot override lived reality.
